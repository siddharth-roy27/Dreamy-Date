## 📖 Overview

**DreamyDate** is not a dating matching app like Tinder or Bumble.

It is a dedicated platform built from the ground up for **couples in long-distance relationships** — providing them with a beautiful, immersive space to connect, share experiences, and keep the romance alive across any distance.

### 🌹 Why DreamyDate?

Long-distance relationships are hard. Video calls alone don't capture the magic of being together. DreamyDate changes that by creating **shared experiences** that feel intimate, interactive, and memorable.

| Traditional Video Calls | DreamyDate |
|:---:|:---:|
| Passive conversations | Interactive activities together |
| No shared context | Synchronized music, movies, games |
| Forgettable moments | AI-generated memory highlights |
| Generic UI | Vintage diary-themed romance aesthetic |
| Just faces on a screen | A world you build together |

### 💫 What Makes It Special

- 🎭 **Interactive Date Experiences** — Not just watching each other, but *doing* things together
- 🎬 **AI-Powered Dream Videos** — Your best moments turned into cinematic love stories
- 📔 **Vintage Diary Aesthetic** — Every interaction feels like writing in a love letter
- 🎵 **Spotify Synchronization** — Dance together, even miles apart
- 📅 **Shared Calendar** — Plan your future together with beautiful date blueprints
- 🧠 **Memory Intelligence** — AI remembers your conversations, quotes, and milestones

### 🤖 AI Dream Video Generator

The crown jewel of DreamyDate. Our **AI pipeline** transforms your date sessions into cinematic, romantic highlight videos you'll want to watch again and again.

#### How It Works

```
Date Session → Memory Extraction → Conversation Analysis →
Romantic Quote Detection → Scene Generation → Cinematic Render →
Memory Library
```

1. **Record** — Captures key moments during your date session
2. **Extract** — AI identifies emotional peaks and special interactions
3. **Analyze** — Conversations are processed with **OpenAI Whisper** for transcription
4. **Detect** — Romantic quotes and meaningful phrases are extracted
5. **Generate** — AI creates visual scenes using **RunwayML** and **Stability AI**
6. **Render** — Scenes are compiled with **FFmpeg** into a cinematic video
7. **Store** — Final video is saved to your Memory Library

## 🚀 Installation & Setup

### Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** (v20 or higher)
- **Bun** (recommended) or npm
- **PostgreSQL** (v16)
- **MongoDB** (v7.x)
- **Redis** (v7.x)

You will also need API credentials for:

- [Spotify Developer Account](https://developer.spotify.com/)
- [Google OAuth Credentials](https://console.cloud.google.com/)
- [OpenAI API Key](https://platform.openai.com/)
- [AWS S3 Bucket](https://aws.amazon.com/s3/)

### Clone Repository

```bash
git clone https://github.com/siddharth-roy27/Dreamy-Date.git
cd Dreamy-Date
```

### Install Dependencies

```bash
bun install
# or
npm install
```

### Environment Variables

Create a `.env` file in the root directory:

```env
# ── Database ──────────────────────────────────────────
DATABASE_URL=postgresql://user:password@localhost:5432/dreamydate
MONGODB_URI=mongodb://localhost:27017/dreamydate_media
REDIS_URL=redis://localhost:6379

# ── Authentication ────────────────────────────────────
JWT_SECRET=your-super-secret-jwt-key-change-in-production
GOOGLE_CLIENT_ID=your-google-client-id
GOOGLE_CLIENT_SECRET=your-google-client-secret

# ── Spotify Integration ───────────────────────────────
SPOTIFY_CLIENT_ID=your-spotify-client-id
SPOTIFY_CLIENT_SECRET=your-spotify-client-secret
SPOTIFY_REDIRECT_URI=http://localhost:3000/auth/spotify/callback

# ── OpenAI / AI Services ─────────────────────────────
OPENAI_API_KEY=sk-your-openai-api-key
RUNWAYML_API_KEY=your-runwayml-api-key
STABILITY_API_KEY=your-stability-ai-key

# ── AWS S3 (Media Storage) ───────────────────────────
AWS_ACCESS_KEY_ID=your-aws-access-key
AWS_SECRET_ACCESS_KEY=your-aws-secret-key
AWS_BUCKET_NAME=dreamydate-media
AWS_REGION=us-east-1

# ── Application ───────────────────────────────────────
VITE_APP_URL=http://localhost:3000
NODE_ENV=development
```

### Run Development Server

```bash
bun dev
# or
npm run dev
```

The application will be available at `http://localhost:3000`.

### Production Build

```bash
bun build
# or
npm run build
```

## 🗄️ Database Schema Overview

### Users

```sql
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email VARCHAR(255) UNIQUE NOT NULL,
  name VARCHAR(100) NOT NULL,
  avatar_url TEXT,
  google_id VARCHAR(255) UNIQUE,
  spotify_connected BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### Partner Links

```sql
CREATE TABLE partner_links (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_a_id UUID REFERENCES users(id),
  user_b_id UUID REFERENCES users(id),
  link_code VARCHAR(12) UNIQUE NOT NULL,
  linked_at TIMESTAMPTZ DEFAULT NOW(),
  anniversary_date DATE
);
```

### Date Sessions

```sql
CREATE TABLE date_sessions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  couple_id UUID REFERENCES partner_links(id),
  type VARCHAR(50) NOT NULL, -- 'hosted' | 'planned'
  activity VARCHAR(100),
  started_at TIMESTAMPTZ DEFAULT NOW(),
  ended_at TIMESTAMPTZ,
  recording_url TEXT,
  ai_video_url TEXT
);
```

### Calendar Entries

```sql
CREATE TABLE calendar_entries (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  couple_id UUID REFERENCES partner_links(id),
  title VARCHAR(255) NOT NULL,
  description TEXT,
  theme VARCHAR(50),
  scheduled_at TIMESTAMPTZ NOT NULL,
  activities TEXT[],
  created_by UUID REFERENCES users(id)
);
```

### Memories

```sql
CREATE TABLE memories (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  couple_id UUID REFERENCES partner_links(id),
  session_id UUID REFERENCES date_sessions(id),
  type VARCHAR(50), -- 'photo' | 'video' | 'note' | 'ai_highlight'
  content_url TEXT,
  caption TEXT,
  tags TEXT[],
  is_favorite BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### Playlists

```sql
CREATE TABLE playlists (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  couple_id UUID REFERENCES partner_links(id),
  name VARCHAR(255) NOT NULL,
  spotify_playlist_id VARCHAR(255),
  tracks JSONB DEFAULT '[]',
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

## 🧭 User Journey

### 1. Creating an Account

Sign up with Google OAuth or email. Set up your profile with your name, photo, and relationship details. You'll receive a unique **partner link code**.

### 2. Linking a Partner

Share your link code with your partner. Once they enter it, your accounts are connected and your shared diary, calendar, and memory library are created.

### 3. Planning a Date

Open the **Diary Calendar**, pick a date and time, choose a theme (Romantic, Adventurous, Cozy, Creative), and add activities. Your partner receives a beautiful invitation.

### 4. Hosting a Date

Click **Host Date** to start an instant video session. Choose activities from the menu or let the AI suggest something based on your mood and history.

### 5. Dancing Together

Enter the **Spotify Dance Hall**, connect your Spotify accounts, pick a playlist, and dance together with synchronized music and ambient visual effects.

### 6. Creating Memories

Every date is automatically saved to your **Memory Library**. Add notes, tag moments, and favorite the highlights. Photos and clips are organized on a beautiful timeline.

### 7. Generating AI Videos

After a date, trigger the **AI Dream Video Generator**. Within minutes, receive a cinematic highlight reel with romantic music, extracted quotes, and AI-generated artwork — ready to share or keep as a cherished memory.

## 🔒 Security

DreamyDate takes your privacy and security seriously.

- 🔐 **OAuth 2.0** — Secure authentication via Google and Spotify
- 🎫 **JWT Tokens** — Short-lived access tokens with refresh rotation
- 🔒 **HTTPS Everywhere** — All traffic encrypted in transit
- 📹 **End-to-End Video Encryption** — Video calls are private by design
- 🗄️ **Encrypted Storage** — All media encrypted at rest in S3
- 🛡️ **Rate Limiting** — Protection against abuse and spam
- 🔍 **Input Sanitization** — SQL injection and XSS prevention
- 🕵️ **Privacy First** — No data sold to third parties, ever
